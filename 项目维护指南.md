# GKD 订阅规则维护指南

## 一、项目概述

### 1.1 什么是 GKD？

GKD（搞快点）是一个基于 Android 无障碍服务的自动化工具，主要用于：
- 自动跳过应用开屏广告
- 关闭应用内弹窗广告
- 自动处理更新提示、青少年模式等干扰弹窗
- 自动执行一些重复性操作

### 1.2 项目关系

```
GKD App (https://github.com/gkd-kit/gkd)
    ↓ 读取订阅规则
本项目 (订阅规则仓库)
    ↓ 定义规则
各个 Android 应用
```

- **GKD App**：安装在 Android 设备上的应用，负责执行规则
- **本项目**：维护规则配置文件，告诉 GKD 如何识别和点击广告
- **规则**：用选择器语法描述界面元素，指导 GKD 自动点击

### 1.3 项目结构

```
subscription/
├── src/
│   ├── apps/              # 各个应用的规则文件
```

## 二、核心概念

### 2.1 规则分类（categories.ts）

```typescript
{
  key: 0,
  name: '开屏广告',
  enable: true,  // 默认启用
}
```

共11个分类：
- **开屏广告**：应用启动时的全屏广告（默认启用）
- **青少年模式**：青少年模式提示弹窗
- **更新提示**：应用更新提醒
- **评价提示**：应用评分请求
- **通知提示**：各种通知弹窗
- **权限提示**：权限请求弹窗
- **局部广告**：页面内的广告卡片
- **全屏广告**：阻碍操作的全屏广告
- **分段广告**：需要多步操作关闭的广告（如朋友圈广告）
- **功能类**：自动执行某些功能
- **其他**：未分类规则

### 2.2 全局规则 vs 应用规则

**全局规则（globalGroups.ts）**
- 适用于所有应用的通用规则
- 例如：通用的"跳过"按钮识别
- 可以设置黑白名单控制哪些应用启用

**应用规则（src/apps/*.ts）**
- 针对特定应用的专属规则
- 例如：微信朋友圈广告、QQ空间广告
- 文件名 = 应用包名.ts

### 2.3 选择器语法

选择器用于定位界面元素，类似 CSS 选择器：

```typescript
// 基础属性匹配
'[text="跳过"]'              // 文本等于"跳过"
'[text*="跳过"]'             // 文本包含"跳过"
'[text^="跳过"]'             // 文本以"跳过"开头
'[text$="跳过"]'             // 文本以"跳过"结尾
'[id="skip_button"]'         // ID匹配
'[clickable=true]'           // 可点击
'[visibleToUser=true]'       // 用户可见

// 尺寸限制
'[width<500 && height<300]'  // 宽度<500且高度<300

// 层级关系
'A > B'                      // B是A的直接子节点
'A < B'                      // A是B的直接子节点
'A + B'                      // B是A的下一个兄弟节点
'A - B'                      // A是B的上一个兄弟节点
'A >n B'                     // B是A的第n代子孙节点

// 逻辑组合
'[text="跳过" || text="Skip"]'  // 或
'[text*="跳过" && width<500]'   // 且
```

### 2.4 规则结构

```typescript
{
  key: 0,                    // 规则唯一标识
  name: '开屏广告',          // 规则名称
  desc: '关闭开屏广告',      // 规则描述
  enable: true,              // 是否默认启用
  fastQuery: true,           // 是否使用快速查询
  matchTime: 10000,          // 匹配时间窗口（毫秒）
  actionMaximum: 2,          // 最大执行次数
  resetMatch: 'app',         // 重置匹配条件
  activityIds: [             // 限定在哪些页面生效
    '.MainActivity',
  ],
  rules: [                   // 具体规则
    {
      key: 0,
      matches: '[text="跳过"]',  // 匹配选择器
      snapshotUrls: [            // 快照URL（用于测试）
        'https://i.gkd.li/i/12345',
      ],
    },
  ],
}
```

## 三、维护工作流程

### 3.1 添加新应用规则

#### 步骤1：创建应用规则文件

```bash
# 文件名 = 应用包名.ts
# 例如：抖音的包名是 com.ss.android.ugc.aweme
touch src/apps/com.ss.android.ugc.aweme.ts
```

#### 步骤2：编写规则

```typescript
import { defineGkdApp } from '@gkd-kit/define';

export default defineGkdApp({
  id: 'com.ss.android.ugc.aweme',  // 应用包名
  name: '抖音',                     // 应用名称
  groups: [
    {
      key: 0,
      name: '开屏广告',
      desc: '关闭抖音开屏广告',
      fastQuery: true,
      matchTime: 10000,
      actionMaximum: 2,
      resetMatch: 'app',
      rules: [
        {
          matches: '[text*="跳过"][text.length<10]',
          snapshotUrls: 'https://i.gkd.li/i/xxxxx',
        },
      ],
    },
  ],
});
```

#### 步骤3：测试规则

```bash
# 检查语法错误
npm run check

# 构建规则文件
npm run build
```

#### 步骤4：提交代码

```bash
git add src/apps/com.ss.android.ugc.aweme.ts
git commit -m "feat(抖音): 添加开屏广告规则"
git push
```

### 3.4 常见问题排查

#### 问题1：规则不生效

**可能原因：**
- 选择器写错了
- activityIds 限制了页面
- matchTime 太短
- 规则被禁用

**排查方法：**
```typescript
// 1. 移除 activityIds 限制
// activityIds: ['.MainActivity'],  // 注释掉

// 2. 延长匹配时间
matchTime: 30000,  // 改为30秒

// 3. 检查 enable 状态
enable: true,

// 4. 使用快照测试选择器
```

#### 问题2：误触其他按钮

**解决方法：**
```typescript
// 使用 excludeMatches 排除误触
excludeMatches: '[text*="搜索"]',

// 添加更多限制条件
matches: '[text="跳过"][width<500][height<300]',

// 限制在特定页面
activityIds: ['.SplashActivity'],
```

#### 问题3：构建失败

```bash
# 检查语法错误
npm run check

# 查看错误信息
npm run build
```

## 四、高级技巧

### 4.1 快速查询优化

```typescript
// 使用 fastQuery 可以提高匹配速度
fastQuery: true,

// 但选择器必须满足条件：
// ✅ 可以快速查询
'[text="跳过"]'
'[id="skip_btn"]'
'[text*="跳过"][clickable=true]'

// ❌ 不能快速查询
'[text~="(?is).*skip.*"]'  // 正则表达式
'A >n B'                    // 不定层级
```

### 4.2 分段规则

用于需要多步操作的场景（如朋友圈广告）：

```typescript
rules: [
  {
    key: 0,
    name: '第一步：点击广告菜单',
    matches: '[text="广告"]',
  },
  {
    key: 1,
    preKeys: [0],  // 必须先执行 key=0
    name: '第二步：点击关闭',
    matches: '[text="关闭"]',
  },
  {
    key: 2,
    preKeys: [1],  // 必须先执行 key=1
    name: '第三步：确认关闭',
    matches: '[text="确认"]',
  },
]
```

### 4.3 位置点击

当元素不可点击时，使用坐标点击：

```typescript
{
  position: {
    left: 'width * 0.9',   // 屏幕宽度的90%
    top: 'height * 0.1',   // 屏幕高度的10%
  },
}
```

### 4.4 全局规则黑白名单

```typescript
// globalDefaultApps.ts
export const openAdBlackListAppIDs = [
  'com.example.app1',  // 禁用全局开屏规则
];

export const openAdWhiteListAppIDs = [
  'com.example.app2',  // 启用全局开屏规则
];
```

## 八、维护建议

### 8.1 规则编写原则

1. **精确匹配**：选择器尽量精确，避免误触
2. **性能优先**：使用 fastQuery，避免复杂选择器
3. **充分测试**：使用快照测试，确保规则正确
4. **文档完善**：添加 desc 和 snapshotUrls

- 使用语义化版本号
- 每次更新记录 CHANGELOG
- 重大变更提前通知用户

### 8.3 社区互动
